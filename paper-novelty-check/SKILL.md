# 去重选题与增量文献检索 Skill

## 目的

接收 SKILL 1 的 References Ledger（文献台账），在后续检索中排除已用文献，确保增量新颖性，避免选题与已有研究重复。

---

## 输入要求

### 必须输入
- **References Ledger**：SKILL 1 输出的文献台账（包含 reference_id, title, first_author, year, url_or_doi, dedupe_key）
- **选题方向**：当前候选题目或研究关键词

### 可选输入
- 学科方向
- 时间范围偏好
- 特定期刊/会议偏好

---

## 执行流程

### Step 1：加载文献台账

从 SKILL 1 的输出中读取 References Ledger，建立已用文献集合。

```
已用文献集合 = {dedupe_key1, dedupe_key2, ..., dedupe_keyN}
```

### Step 2：增量检索

基于选题方向，在以下平台进行**增量检索**（排除已用文献）：

1. **Google Scholar**：使用 `site:scholar.google.com` + 关键词
2. **Semantic Scholar**：使用 API 或网页搜索
3. **arXiv**：针对计算机科学等领域
4. **PubMed**：针对生物医学领域
5. **Web of Science / Scopus**：如可访问

**检索策略**：
- 在检索式中增加时间限定（如近2年），获取最新研究
- 使用 SKILL 1 未覆盖的关键词组合
- 关注综述文章的"未来研究方向"部分

### Step 3：去重比对

将检索结果与已用文献集合进行比对：

```
对于每篇新文献：
  计算 dedupe_key（优先 DOI，其次 title+first_author+year）
  如果 dedupe_key 在已用文献集合中 → 标记为 [已收录]，跳过
  否则 → 标记为 [新增]，纳入候选
```

### Step 4：新颖性评估

对筛选出的新增文献进行分析：

| 评估维度 | 说明 |
|---------|------|
| **选题重合度** | 新增文献与候选题目的研究问题重叠程度 |
| **方法重合度** | 新增文献使用的方法与候选方案的相似度 |
| **结论一致性** | 新增文献的结论是否已经回答了候选研究问题 |
| **时间新颖性** | 新增文献的发表时间（越近越需要关注） |

### Step 5：输出结果

#### 5.1 去重检索报告

```
| 检索平台 | 检索式 | 总结果 | 已收录 | 新增 | 高相关新增 |
|---------|--------|--------|--------|------|-----------|
| Google Scholar | ... | 50 | 35 | 15 | 3 |
| Semantic Scholar | ... | 30 | 20 | 10 | 2 |
...
```

#### 5.2 新颖性判定

| 判定 | 条件 |
|------|------|
| ✅ **高新颖性** | 无高相关新增文献，选题空间充足 |
| ⚠️ **中等新颖性** | 有少量高相关新增文献，需微调选题角度 |
| ❌ **低新颖性** | 有大量高相关新增文献，建议重新选题 |

#### 5.3 新增核心文献列表

```
| reference_id | title | first_author | year | url_or_doi | dedupe_key | 相关性 |
|-------------|-------|-------------|------|------------|------------|--------|
| N1          | ...   | ...         | 2025 | https://...| DOI:10.xxx | 高 |
...
```

#### 5.4 更新后的 References Ledger

将新增文献追加到原 References Ledger，输出更新后的完整台账（供后续 SKILL 3 或再次迭代使用）。

---

## 与 SKILL 1 的协作协议

### 接收输入
```
SKILL 1 输出 → References Ledger (SKILL 1)
              → 候选题目
              → 研究关键词
```

### 输出传递
```
SKILL 2 输出 → 更新后的 References Ledger → SKILL 3 或 SKILL 1 迭代
              → 新颖性判定报告
              → 新增核心文献列表
```

### 迭代循环
如果新颖性判定为"低新颖性"，建议：
1. 返回 SKILL 1 重新生成候选题（使用新的关键词方向）
2. 再次执行 SKILL 2 验证
