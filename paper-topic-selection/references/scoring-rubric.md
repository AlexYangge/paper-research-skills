# 四维评分详细量规

## 评分总则

每个维度 1-5 分，评分标准如下。评分时需要给出**具体理由**，不能只给数字。

---

## 维度一：价值（理论/实践意义）

| 分值 | 标准 | 典型描述 |
|------|------|---------|
| 5 | 高价值 | 填补重要理论空白，或解决行业关键痛点，成果可直接应用 |
|  |  | 已有明确证据显示该问题的重要性（如政策文件、行业报告引用） |
| 4 | 有价值 | 对现有理论有推进，或对实践有参考意义 |
| 3 | 一般 | 有一定价值，但贡献边际较窄 |
| 2 | 较低 | 价值不明确，或已有大量类似研究 |
| 1 | 很低 | 几乎看不到理论或实践价值 |

**评分锚点：**
- 是否有明确的需求信号（文献反复提及、政策关注、行业痛点）？
- 解决这个问题后，谁会受益？受益多大？

---

## 维度二：创新（可证明的新意）

| 分值 | 标准 | 典型描述 |
|------|------|---------|
| 5 | 显著创新 | 提出新理论/新方法/新视角，且有初步证据支持 |
| 4 | 明显创新 | 在现有研究基础上做出可辨识的推进 |
| 3 | 有限创新 | 对已有方法的改进或组合，有一定新意 |
| 2 | 微小创新 | 主要是应用已有方法到新场景 |
| 1 | 无创新 | 完全重复已有工作 |

**⚠️ 创新≠"首次研究"**
- 可接受的新意类型：新应用场景、新数据集、方法改进、跨学科整合
- 不可接受的创新声明："之前没有人研究过"（必须提供文献检索证据）

---

## 维度三：可行（数据、方法、周期）

| 分值 | 标准 | 典型描述 |
|------|------|---------|
| 5 | 高度可行 | 数据可公开获取，方法成熟，周期内可完成 |
| 4 | 基本可行 | 主要条件满足，个别环节需要额外努力 |
| 3 | 勉强可行 | 存在明显障碍但有替代方案 |
| 2 | 较难可行 | 多个关键环节存在不确定性 |
| 1 | 不可行 | 核心数据不可获取或方法超出能力范围 |

**可行性检查清单：**
- [ ] 数据来源明确且可获取（A/B/C层至少一项）
- [ ] 方法已在类似研究中成功使用
- [ ] 所需软硬件资源可获得
- [ ] 研究周期与论文截止时间匹配

---

## 维度四：匹配（能力、导师、目标）

| 分值 | 标准 | 典型描述 |
|------|------|---------|
| 5 | 完美匹配 | 完全符合用户能力、导师方向和未来目标 |
| 4 | 较好匹配 | 主要方面匹配，个别需要学习 |
| 3 | 基本匹配 | 部分匹配，需要较多额外学习 |
| 2 | 较难匹配 | 多个方面不匹配 |
| 1 | 很不匹配 | 与用户背景严重不符 |

**匹配度评估要素：**
- 方法与用户已掌握技能的匹配度
- 方向与导师研究领域的匹配度
- 课题对未来目标（就业/申博/发文）的支持度

---

## 综合评分计算

```
综合分 = 价值 × w1 + 创新 × w2 + 可行 × w3 + 匹配 × w4

默认权重（均等）：w1=w2=w3=w4=1
推荐权重（可行性优先）：w1=1, w2=1, w3=1.5, w4=1
```

**权重调整建议：**
- 硕士论文：可行性权重可提高（按时毕业优先）
- 博士论文/发文导向：创新性权重可提高
- 应用型研究：价值权重可提高
